Detailed Description

The metadata "header" accompanying any heap-allocated object.

This header should be embedded in heap-allocated objects as the first field (at memory offset 0).

The kind field is used to indicate what ssm_kind an object is, and thus what its size and memory layout are.

Note
The interpretation and usage of the ref_count and tag fields may depend on the value of kind. For instance, some object kinds don't use one or both of those fields, while others may use them together as a single 16-bit field.
